# 제품 페이지 스펙 (단일 진실 소스) — looky.my

> 정책·엔티티는 `domain.md`, 결과 알고리즘 상세는 루트 `기획.md`. 이 문서는 **페이지 단위**: 목적 / 역할 분기 / 보여질 데이터 / 화면 항목 / 상태 3종 / 디자인 방향 / 분석 이벤트.
> wireframe-builder·flow-reviewer·frontend-dev가 참조. **MVP: 비회원·재설문 없음·참여자 3명 응답 시 단일 결과 페이지 생성(시간 제한 없음, 2026-07-05 확정)·디자인 가이드 초기 미비(저충실도부터).**

## 엔티티 (요약 — 상세는 domain.md)
- **주인공(Creator)**: 닉네임(기기 로컬스토리지), **고유 링크 토큰**(추측 불가), 생성 시각, 본인 설문 응답. 서버 계정 없음.
- **설문 응답(Response)**: 어느 링크(주인공)에 대한 것인지, 응답자 **익명**(여러 명/한 명 다회 **구분 안 함**), 8문항 답변, 생성 시각.
- **결과(Result)**: 주인공별, 조하리 4칸 형용사 집합, **인생네컷 4컷**(①②③ AI 생성 / ④ 등 빈 칸 고정 이미지), 설명문.
- **문항(Question)**: 200개 풀(4성향×50), 5지선다 상황극, 성향 태그 내재화(비노출). 출제는 풀에서 **랜덤 8개**.

## 역할 2-뷰 (회원 구분 아님 — 행위 구분)
- **생성자(주인공)** — 닉네임 설정 → 자기 설문(8문항) → 링크 생성·공유 → 참여자 응답 3건 채워지면 즉시 자기 결과 열람(링크가 열쇠).
- **참여자** — 주인공 링크로 진입 → 그 사람에 대한 설문(8문항) → 제출. 신원·계정 없음.

## 상태머신 (링크 = 단일 URL이 전환, 시간 제한 없음 — 2026-07-05 확정)
```
링크 생성 → [설문 수집 · 기간 제한 없음]  ──응답 3건 확보──▶ [결과: 인생네컷 + 설명문]  (단일 페이지)
                                        ──AI 결과 생성 실패──▶ [재시도 유도 view]
              (링크 만료/없음)          ─────────────────▶ [만료·404 안내 view]
```

## 페이지 인벤토리 (MVP)
1. 랜딩 / 진입 ✅상세
2. 닉네임 설정 (주인공 온보딩 1) ✅상세
3. 자기 설문 (온보딩 2 · 필수 선행) ✅상세
4. 링크 공유 페이지 (수집 중) ✅상세
5. 참여자 설문 ✅상세
6. 결과 페이지 (인생네컷 + 설명문) ✅상세
7. 결과 생성 실패 — 재시도 view 🔸스텁
8. 만료 / 없는 링크 view 🔸스텁

---

# 1. 랜딩 / 진입 ✅

**목적**: 서비스 소개 + 주인공 시작 유도. "친구들이 본 나를 인생네컷으로."

**역할 분기**: 신규 방문 → 랜딩. (참여자는 보통 공유 링크로 바로 5번 진입.) 로컬스토리지에 진행 중인 내 링크가 있으면 이어가기 노출.

**보여질 데이터**: 컨셉 한 줄 카피, 예시 인생네컷 비주얼.

**화면 항목**: 로고/타이틀, 컨셉 카피, [시작하기] 단일 CTA, (있으면) [내 결과/진행 이어보기].

**상태 3종**: 로딩=해당 적음 / 에러=리소스 로드 실패 재시도 / 빈=해당 없음.

**디자인 방향**: 10~20대 여성 감성, 인생네컷 2×2 프리뷰 중심 히어로 (저충실도는 회색 박스).

**분석 이벤트**: `landing_view`, `start_click`, `resume_click`.

---

# 2. 닉네임 설정 ✅

**목적**: 주인공 식별의 시작. 닉네임을 **로컬스토리지에 저장**(서버 계정 아님).

**역할 분기**: 주인공만. 시작하기 직후.

**화면 항목**: 닉네임 입력(길이 제한 `TODO(✍️)`), [다음] → 자기 설문.

**상태 3종**: 로딩=거의 없음 / 에러=입력 규칙 위반 인라인 / 빈=해당 없음.

**디자인 방향**: 폼 1열, 모바일 퍼스트.

**분석 이벤트**: `nickname_view`, `nickname_submit`.

**TODO(✍️)**: 닉네임 길이·금칙어 규칙(중복 검사는 계정이 없어 불필요).

---

# 3. 자기 설문 (필수 선행) ✅

**목적**: 조하리 "나 vs 친구" 비교의 본인 쪽 데이터. **링크 생성 전 반드시 1회.**

**역할 분기**: 주인공만. 온보딩 단계.

**보여질 데이터**: 풀에서 **랜덤 8문항**(5지선다 상황극, 주어 "나"로 치환). 성향 태그·점수 **비노출**.

**화면 항목**: 진행 표시(n/8), 문항+5보기, **"잘 모르겠다" 중립 선택지**, [다음/완료].

**상태 3종**: 로딩=문항 로드 / 에러=저장 실패 재시도 / 빈=해당 없음.

**디자인 방향**: 한 화면 한 문항, 탭하기 쉬운 큰 보기 버튼(모바일).

**분석 이벤트**: `selfsurvey_start`, `selfsurvey_q{n}`, `selfsurvey_complete`. ★ `자기설문 완료→링크공유` 퍼널.

---

# 4. 링크 공유 페이지 (수집 중) ✅

**목적**: 핵심 루프. 고유 링크를 **인스타 스토리·카톡으로 퍼뜨려** 참여자를 모은다.

**역할 분기**: 주인공(자기 링크 관리 뷰). 응답 수 표시(시간 제한 없음 — 3건 채워질 때까지 계속 이 화면).

**보여질 데이터**: 내 고유 링크, 모인 응답 수(3건 목표 게이지).

**화면 항목**
- **[인스타 스토리 공유]** — 서비스 이미지를 공유 시트로, **링크는 클립보드 복사**(스토리엔 링크 첨부 제약 회피).
- **[카카오톡 공유]**, [링크 복사].
- 응답 수 게이지 / "3명 모이면 더 정확해져요" 안내.
- (응답 3건 확보 시) → 결과(6)로 자동 전환.

**상태 3종**: 로딩=링크 발급(즉시) / 에러=공유 실패 시 복사 fallback / 빈=아직 응답 0 → "첫 친구를 기다리는 중".

**디자인 방향**: 공유 CTA 강조, 응답 수 게이지 시각화. 스토리용 이미지 에셋 필요(`design-guide`).

**분석 이벤트**: `share_view`, `share_story_click`, `share_kakao_click`, `link_copy`, `respondent_count_{n}`. ★ `링크공유→참여완료` 핵심 퍼널.

# 5. 참여자 설문 ✅

**목적**: 친구가 주인공에 대한 인식을 남긴다. 유입 → 참여 전환이 서비스 생존선.

**역할 분기**: 참여자(공유 링크 진입). 신원·로그인 없음. 만료된 링크면 8번으로.

**보여질 데이터**: "{닉네임}은 어떤 사람일까?" + **랜덤 8문항**(주어 "이 사람"). 본인 설문과 문항 달라도 됨.

**화면 항목**: 주인공 닉네임 헤더, 진행(n/8), 문항+5보기, [완료] → 감사 화면 + **나도 만들기 CTA**(바이럴 루프).

**상태 3종**: 로딩=문항 로드 / 에러=제출 실패 재시도 / 빈=만료 링크 → 8번 전환.

**디자인 방향**: 3번과 동일 설문 UX. 완료 후 "나도 해보기"로 신규 주인공 전환.

**분석 이벤트**: `respond_view`, `respond_q{n}`, `respond_complete`, `respond_to_create_click`.

# 6. 결과 페이지 (인생네컷 + 설명문) ✅

**목적**: 참여자 응답 3건이 채워지는 즉시 자동 생성되는 **단일 결과 페이지**(시간 제한 없음). 재공유로 루프 재점화.

**역할 분기**: 누구나(링크가 열쇠). 주인공이 주 소비자.

**보여질 데이터**: 조하리 **2×2 인생네컷**(①모두가 아는 나 ②친구만 아는 나 ③나만 아는 나 ④아직 모르는 나 — 라벨 단일 소스 `data/quadrants.ts`), 칸별 형용사, 설명문("스스로는 신중하다지만 친구들은 모험가로 봤어").

**화면 항목**
- 인생네컷 4컷 그리드 + 칸 라벨/형용사.
- **빈 칸(주로 ④)**: 고정 이미지(안개·물음표) + "아직 너를 다 발견하지 못했어 — 다른 사람들에게도 물어봐" 재참여 메시지.
- 설명문 영역, **[다시 공유하기]**(재참여 유도), [이미지 저장].

**상태 3종**: **로딩=이미지 생성 대기**(가장 중요 — 진행 표현 필수) / 에러=생성 실패 재시도 / 빈=칸 비면 고정 이미지로 대체(빈 화면 아님).

**디자인 방향**: 인생네컷 = 폴라로이드/필름 느낌 4컷, 캡처·공유 좋은 비율. 형용사 톤은 **긍정/중립만**(비하·외모·직설 부정 금지 — `domain.md §1`).

**분석 이벤트**: `result_view`, `result_image_ready`, `result_reshare_click`, `result_save_click`. ★ `결과 도달→재공유` 퍼널.

# 7~8. (스텁)
